# 업무와 일상의 경계를 긋는 커뮤니케이션

슬랙 알림 설정과 업무 시간 외 연락 차단 전략을 통해 개인의 휴식 시간을 물리적으로 보호하는 실무적인 가이드를 제공합니다.

## 개요

디지털 협업 도구의 발달은 언제 어디서나 소통할 수 있는 편리함을 주었지만, 동시에 업무와 일상의 경계를 무너뜨리는 부작용을 낳기도 했습니다. 지속 가능한 성장을 위해서는 '연결되지 않을 권리'를 확보하는 것이 필수적입니다. 본 문서에서는 슬랙(Slack)을 중심으로 업무 시간 외의 방해를 차단하고 집중력을 높이는 방법을 알아봅니다.

## 1. 슬랙 알림 스케줄 설정 (Notification Schedule)

업무 시간이 종료된 후 자동으로 알림이 오지 않도록 설정하여 물리적인 휴식 시간을 확보하십시오.

1. 슬랙 상단 프로필 이미지를 클릭한 후 **[기본 설정(Preferences)]**으로 이동합니다.
2. **[알림(Notifications)]** 탭을 선택합니다.
3. **[알림 스케줄(Notification schedule)]** 섹션에서 다음을 설정합니다.
   - **요일**: `월요일~금요일` 또는 `매일` 선택
   - **시간**: 본인의 업무 시간(예: `오전 9:00 ~ 오후 6:00`) 지정

이렇게 설정하면 지정된 시간 외에는 데스크톱과 모바일 기기로 푸시 알림이 전송되지 않으며, 상태 아이콘이 '방해 금지 모드(Zzz)'로 변경됩니다.

## 2. 상태 메시지와 이모지 활용

동료들이 현재 나의 상태를 인지할 수 있도록 명확한 메시지를 남기면 불필요한 재촉을 방지할 수 있습니다.

### 권장되는 상태 설정 예시
- 🌴 **휴가 중**: `12/20 ~ 12/22 휴가 (급한 용무는 @팀장님께)`
- 🎧 **집중 근무**: `오후 2시까지 기획안 작성 중 (급한 건은 DM)`
- 🏃 **외근/이동 중**: `외부 미팅 중, 확인이 늦을 수 있습니다`
- 🌙 **업무 종료**: `업무 종료, 내일 오전 9시 이후 확인합니다`

## 3. 예약 전송 기능을 통한 배려

늦은 밤이나 주말에 갑자기 업무 아이디어가 떠올랐다면, 즉시 메시지를 보내는 대신 **예약 전송** 기능을 활용하세요.

```text
[메시지 예약 전송 방법]
1. 메시지 입력창에 내용을 작성합니다.
2. [보내기] 버튼 옆의 작은 화살표(∨)를 클릭합니다.
3. [내일 오전 9:00에 전송] 또는 [사용자 지정 시간]을 선택합니다.
```

이 기능을 사용하면 메시지는 다음 날 업무 시간에 맞춰 전송되어 동료의 휴식 시간을 방해하지 않으면서도 내용을 잊지 않고 전달할 수 있습니다.

## 4. 긴급 상황 프로토콜 구축

팀 내에서 '긴급함'에 대한 정의를 미리 합의하는 것이 중요합니다. 모든 메시지가 긴급한 것은 아닙니다.

- **긴급(Critical)**: 서버 다운, 보안 이슈 등 (전화 연락 허용)
- **보통(Normal)**: 다음 영업일 오전까지 처리 가능한 업무 (슬랙 메시지 기록)
- **낮음(Low)**: 단순 공유 사항 (스레드 또는 공개 채널에 업로드)

> 💡 **Tip**: 슬랙의 '방해 금지 모드'를 뚫고 강제 알림을 보내는 기능은 정말 생사가 걸린 긴급 상황에서만 사용하도록 팀 규칙을 정하십시오.

## 결론

명확한 커뮤니케이션 경계 설정은 이기적인 행동이 아니라, 장기적으로 팀의 생산성을 높이고 구성원의 번아웃을 예방하는 전문적인 태도입니다. 오늘부터 나만의 알림 스케줄을 설정하고 팀원들과 공유해 보세요.
